# World Model

The world model describes the small set of identity decisions a football register must
make before matching providers:

- What counts as an entity?
- Which identifiers are stable?
- Which upstream fields are identity evidence?
- How do provider mappings age?
- Which weak matches require human review?
- How are competitions, seasons, stages, and matches represented?

These pages define concepts. The more prescriptive Reep-style recommendations live in
[practice guides](../guides/README.md). This split keeps provider and world-model
reference material easy to contribute to while still giving maintainers clear doctrine
where judgement matters.
